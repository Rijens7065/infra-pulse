What we do

Three layers, one autonomous loop.

Every CloudSentro system follows the same pattern: an ML model produces a structured signal, an agent reasons about it with typed tools, and a human approves the fix through GitOps. No mystery, no magic — just well- composed pieces.

Detect

Models, not vibes

An LSTM autoencoder learns the cluster's normal heartbeat. A classifier labels deviations into six failure modes with a confidence score. Cheap, fast, and explainable.

Reason

Claude with tools

When confidence exceeds 75%, Claude runs a tool-use loop — pulls metrics, reads the actual Terraform file, lists recent commits — then writes a step-by-step reasoning chain you can audit.

Ship

Real GitHub PRs

The agent opens a real pull request with the change, rollback plan, and reasoning trail. A human approves. HCP Cloud applies. The infrastructure heals.

A typical incident

2:47 AM. Memory leak. Asleep.

Here's what happens in a normal CloudSentro-watched cluster when a pod starts misbehaving — no human touched a keyboard until the very last step.

t=0s

Anomaly enters the cluster

A pod's memory RSS starts climbing toward its limit. Real or injected — the loop doesn't care.

t=30s

ML model classifies it

anomaly_score: 0.95, failure_mode: OOM_LEAK, confidence: 0.88 — emitted from /predict and scraped by Prometheus.

t=5min

Agent investigates

Reads metrics, lists Kubernetes events, looks at recent infra commits, fetches the relevant Terraform file.

t=5min 20s

Pull request appears

Branch fix/agent-20260510-oom_leak. Diff, reasoning chain, rollback. Labels: agent-remediation, oom_leak.

+ human review

You wake up to one decision

Read the reasoning. Approve. HCP Cloud applies. The cluster heals.
