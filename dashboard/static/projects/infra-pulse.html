Architecture

The three layers

Each layer does one thing well, and the contract between them is small and typed. The ML pod emits an AnomalySignal; the agent emits a RemediationPlan; humans emit "merge" or "close."

Layer 1 · Detect

ML pod

PyTorch LSTM autoencoder reconstructs a 60-minute × 7-channel window. Per-channel reconstruction error fed to a two-stage classifier (IsolationForest gate + RandomForest).

Output: {score, mode, confidence, explanation}
Latency: <200ms inference
Accuracy: 99.23% on held-out 15% split

Layer 2 · Reason

Claude agent

Polls every 300s. If signal warrants it, runs a tool-use loop with Claude — up to 10 turns. Reads recent infra commits, lists Kubernetes events, reads the relevant Terraform file before proposing changes.

Model: claude-sonnet-4-20250514
Tools: 7 typed tools
Output: RemediationPlan + GitHub PR

Layer 3 · Ship

GitOps gate

Every change reaches infrastructure as a real GitHub PR with five mandatory sections. A human approves. HCP Cloud applies. GitHub Actions polls HCP Cloud's API and reflects the apply outcome on the commit status.

Approval: manual, always
Apply: HCP Cloud, OIDC-authenticated
Status: truthful — green only when apply succeeded

Deep dive

Pick a thread

Zero static credentials, anywhere

GitHub Actions authenticates to Azure via OIDC federated credentials, assuming a managed identity scoped to AcrPush. HCP Cloud authenticates to Azure via a different OIDC chain — a separate app registration with broader Terraform-applier scope. The two identities are never confused (that was a real bug we hit and fixed by splitting the variable).

The only static secrets in the entire system live in Azure Key Vault: the Anthropic API key, the GitHub App private key, and the auto-generated Grafana admin password. Pods read them at runtime via DefaultAzureCredential.

Every pod runs as non-root, with a read-only root filesystem, capabilities.drop: ["ALL"], and the RuntimeDefault seccomp profile.

What the model actually learns

We train an LSTM autoencoder on synthetic 30-day, 7-channel AKS metrics. The encoder squeezes a 60-minute window down to a 32-dimensional latent vector; the decoder reconstructs the window. Reconstruction error on the last 10 timesteps is the anomaly score.

Then a two-stage classifier sits on top: IsolationForest as a binary anomalous-or-not gate, then RandomForest to label the six failure modes — NORMAL, OOM_LEAK, CPU_THROTTLE, NETWORK_DEGRADATION, COST_SPIKE, SECURITY_DRIFT.

Test accuracy on the held-out 15% split: 99.23%. CPU inference: under 200ms on Standard_D2s_v3.

Why an agent on top of the model

The model is a good classifier but a terrible diagnostician. "OOM_LEAK at 0.95 confidence" tells you a class — it doesn't tell you which Terraform file to change, whether someone shipped a related fix in the last hour, or what the right rollback path looks like.

Claude runs a tool-use loop, bounded to 10 turns, with seven typed tools: get_current_anomaly_signal, get_azure_monitor_metrics, get_recent_infra_changes, get_kubernetes_events, read_terraform_file, create_remediation_pr, and log_audit_event.

The system prompt enforces six hard rules. The tool code enforces the same rules, with unit tests, in case the prompt drifts. Belt, suspenders.

How $37 a month is achievable

Single-node AKS cluster on Standard_D2s_v3 (~$70 on-demand, cheaper if scaled to zero overnight). Basic ACR (~$5). Free AKS control plane. Cloudflare DNS + TLS (free). Azure Blob audit log (cents). HCP Cloud free tier. Claude API at roughly 500 agent calls/month (~$6).

A hard $50/month Azure budget alert sits at 70% warn and 90% critical. The agent has an explicit hard rule that it cannot propose changes to the budget module — preventing a feedback loop where the agent helpfully raises its own ceiling.
