How we work

Principles

A few commitments that show up in every CloudSentro project.

01

Zero static secrets

OIDC federated credentials end-to-end. No client secrets in CI, no service-account keys in pods, no PATs in workflows. If a secret needs to exist, it lives in Key Vault and only one workload identity can read it.

02

Models do the easy work

An LLM is a wonderful generalist but an expensive classifier. We put a cheap, fast, accurate model in front of the agent and only invoke Claude when there's structured signal worth reasoning about.

03

Humans approve, machines apply

The agent proposes. The human approves. The pipeline applies. Every change touches infrastructure only after passing through a GitOps gate. "Autonomous" doesn't mean "unsupervised."

04

Bounded by code, not vibes

Agent hard rules are enforced in both the system prompt and the tool code, with unit tests. "Don't write outside infra/" isn't a polite request — it's a regex check that fails the tool call.
